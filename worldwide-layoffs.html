Worldwide Layoffs Analysis

View on GitHub

The Challenge

I inherited a messy layoffs.csv—duplicates, inconsistent categories, blank dates, and mixed formats—yet stakeholders urgently needed clear trends on who, where, and when layoffs hit hardest.

My Approach

  • Created a staging table and used window functions to identify and remove duplicate rows (World_layoffs_Data_Cleaning.sql).
  • Standardized categorical fields—trimming whitespace, unifying industry names, correcting misspellings, and converting date strings to proper DATE types.
  • Replaced blanks with NULL, then backfilled missing industry values via self-joins—ensuring no data gaps remained.
  • Ran exploratory SQL (World_layoffs_EDA.sql) to surface top companies, industries, and countries by total layoffs, built rolling monthly totals, and ranked yearly leaders.

Results & Impact

The cleaned dataset powered a rich EDA—revealing the top-5 most impacted companies each year, identifying crypto and tech as the hardest-hit industries, and mapping global layoff waves over time. This work not only honed my SQL mastery (CTEs, window functions, date transforms) but gave leadership the real-time insights they needed to monitor workforce trends and make proactive decisions.
